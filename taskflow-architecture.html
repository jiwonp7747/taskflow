ğŸ”— External Integrations

AI Agent Integration (Claude Code)

TaskFlow integrates with Claude Code CLI for AI task execution:

Execution Method

  • Binary: claude CLI (must be installed)
  • Working Dir: Configurable, defaults to cwd
  • Timeout: 10 minutes (600s) per task
  • Output Capture: stdout/stderr piped to logs
  • Concurrency: Sequential (1 at a time)

Task Handoff

  • Input: Full markdown file content
  • Context: Task ID, title, requirements
  • Feedback: "Feedback" section for refinement
  • Output: Updates "AI Work Log" section
  • Status Change: TODO â†’ IN_PROGRESS â†’ IN_REVIEW

Helper Script Integration

CLI commands for AI agents to interact with TaskFlow:

Command Usage Description
npm run ai:list Get eligible tasks Lists all tasks with assignee:ai-agent and status:TODO/NEED_FIX
npm run ai -- show [id] Show task details Display full task content including requirements and feedback
npm run ai -- start [id] Start task Change status from TODO to IN_PROGRESS
npm run ai -- complete [id] "[msg]" Complete task Change status to IN_REVIEW and append message to AI Work Log

ğŸ“‹ Architecture Summary

Design Pattern

Hexagonal Architecture with Domain-Driven Design principles. Clear separation of concerns across presentation, application, domain, and infrastructure layers.

Data Model

Git-friendly markdown files with YAML frontmatter. Each task is a self-contained file supporting rich descriptions, requirements, feedback, and AI work logs.

Integration Model

File-system based integration with Claude Code CLI. Real-time synchronization between file changes and UI. No external databases or APIs required.

State Management

React hooks for frontend state (useTasks, useConfig, useAIWorker). Server maintains AI Worker state with WebSocket/SSE broadcasting to connected clients.

Development Workflow

TypeScript + Next.js + React for modern development. Modular component structure with clear separation of concerns. Easy to extend and maintain.

Deployment Model

Single-user, local-first deployment. Runs on developer's machine via npm run dev. Can be packaged as Electron app in future. No server infrastructure needed.
